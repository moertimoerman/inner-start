# Feature Request Playbook

## Flow

1. Repo Auditor checks current implementation and adjacent systems.
2. Decision Guard checks locked decisions.
3. Product & Conversion Agent reviews if the feature affects product, copy, pricing, onboarding, or positioning.
4. Builder implements the smallest viable change.
5. QA / Safety Checker reviews regressions and drift.
6. Update reports if the change affects architecture or MVP scope.

## Focus

- Solve the actual request
- Avoid infrastructure inflation
- Keep standard MVP cheap and predictable
- Keep future premium support isolated
